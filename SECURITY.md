# Security policy

## Supported versions

Security fixes are provided for the latest released minor version. Users should
upgrade to the newest patch before reporting an issue that may already be fixed.

## Reporting a vulnerability

Do not disclose suspected vulnerabilities in a public issue, discussion, pull
request, or chat transcript.

Use GitHub's private vulnerability reporting for this repository when it is
available: open the **Security** tab, choose **Advisories**, then **Report a
vulnerability**. Include the affected version, impact, reproduction steps, and
any suggested mitigation.

If private reporting is not available, open a public issue asking the repository
owner to establish a private contact channel. Do not include security-sensitive
technical details in that issue.

Reports will be acknowledged after review. Please allow time to reproduce,
prepare a coordinated fix, and publish updated TypeScript and Python artifacts
before public disclosure.
